# 王者荣耀生日趣味测评网页 PRD

## 一、项目概述

一个面向朋友生日祝福的王者荣耀趣味测评网页。用户通过填写游戏信息、完成趣味选择题并抽取运势签文，获得专属五维评分、峡谷人设评语和生日彩蛋祝福。网站整体强调轻松、好玩、专属感和生日氛围，不追求严肃测评，而是通过游戏化互动制造惊喜感。

本项目的目标是打造一套能让朋友在生日当天通过小游戏获得专属祝福与角色人设的互动体验。它不仅是测评工具，更是专属生日彩蛋场景，强调“这是为你定制的”“这是我们之间的回忆”。

## 二、目标用户与体验定位

- 目标用户：喜欢王者荣耀、习惯与朋友开黑的年轻玩家，尤其适合生日当天为朋友准备惊喜互动的人群。
- 体验定位：轻松、趣味、生日气氛浓厚、结果有专属感。
- 成功指标：用户能够顺畅完成“填信息→答题→抽签→评分→人设评语→生日彩蛋”的流程；结果让用户感到有趣、符合个人风格；页面视觉轻盈、生日元素明显；单题模式避免用户分心。

## 三、核心流程

1. 填信息
2. 答题
3. 抽签
4. 评分
5. 人设评语
6. 生日彩蛋

每一页只会出现一个问题，用户回答完一个问题之后才会跳到下一个。整个流程不允许用户返回前面回答的问题。

## 四、页面输入项与初始画像规则

### 4.1 输入项

- 我们第一次见面，我用的是什么英雄？
  - 这是一个隐藏彩蛋问题。
  - 用户填写答案后，如果答对则弹出“答对啦！看来这段峡谷记忆你还记得。专属生日测评正式开启！”
  - 答错则弹出“好吧，记错也没关系，毕竟重要的是后来我们一起玩过的每一局。生日测评继续开启！”
  - 该问题不建议强行影响核心评分，而是用于增强专属感和后续彩蛋文案。

- 请输入你的峡谷大名
  - 用户昵称输入框，用于后续结果页展示和文案呈现。

- 请输入你的本命英雄，例如：李白、瑶、貂蝉、马可波罗
  - 本命英雄将影响操作分、意识分、队友好感度、人设标签。

- 常用分路
  - 选项：打野 / 中单 / 对抗路 / 发育路 / 游走 / 全能补位
  - 影响：操作分、意识分、carry 倾向、团队倾向。

- 游戏作息
  - 选项：深夜上分党 / 午休摸鱼党 / 放学下班党 / 周末集中党 / 随时在线党
  - 影响：结果文案的语气与角色形象色彩，辅助生日祝福风格，不作为主要评分维度。

- 游戏性格
  - 选项：认真指挥型 / 安静发育型 / 温柔稳局型 / 整活气氛型 / 直爽吐槽型 / 佛系随缘型
  - 影响：心态分、摆烂指数、队友好感度。

- 组队状态
  - 选项：固定双排 / 三五排开黑 / 常年单排 / 随缘组队 / 陪朋友上线 / 看心情，都可以
  - 影响：队友好感度、人设类型。

- 游戏 ID
  - 通过关键词做趣味标签，但不影响核心评分。
  - 该项旨在增强专属感，例如“ID 里有‘奶’、‘爷’、‘骚’、‘狗’ 等关键词可产生对应轻松标签”。

### 4.2 初始画像原则

基础信息不是装饰，而是先给用户一个“初始画像”。

- 本命英雄：影响操作分、意识分、队友好感度、人设标签；
- 常用分路：影响操作分、意识分、carry 倾向、团队倾向；
- 游戏性格：影响心态分、摆烂指数、队友好感度；
- 组队状态：影响队友好感度和人设；
- 游戏 ID：可以通过关键词做趣味标签，但不要影响太大；
- 第一次见面英雄：可以做隐藏彩蛋，不建议强行影响核心评分。

## 五、测评选择题与倾向规则

测评题目共有 6 题，每题选项 A/B/C/D 对应不同倾向。所有题目采用单题页面模式，用户选择后直接进入下一题。

### 5.1 题目与对应倾向

题目 1：逆风局你的第一反应？
- A. 先稳住发育，找机会慢慢翻
  - 对应倾向：稳健、意识、心态
- B. 开始观察局势，试图指挥队友
  - 对应倾向：指挥、carry、认真
- C. 心态有点波动，但还能继续打
  - 对应倾向：随缘、摆烂
- D. 先安抚队友，别急着内讧
  - 对应倾向：温柔、队友好感度

题目 2：拿到人头或打出高光之后，你会？
- A. 赶紧推塔拿龙，扩大优势
  - 对应倾向：意识、carry
- B. 小小得意一下，但继续认真打
  - 对应倾向：操作、稳健
- C. 有点上头，忍不住继续追
  - 对应倾向：操作、上头、摆烂风险
- D. 发个信号或表情，和队友分享快乐
  - 对应倾向：整活、队友好感度

题目 3：队友抢了你的 Buff，你会？
- A. 算了算了，先赢再说
  - 对应倾向：心态、队友好感度
- B. 默默走开，假装无事发生
  - 对应倾向：安静、独狼
- C. 发个问号，表达一下态度
  - 对应倾向：直爽、吐槽
- D. 直接沟通：下次能不能别抢
  - 对应倾向：认真、指挥

题目 4：排位最害怕遇到哪种情况？
- A. 队友挂机、摆烂、乱送
  - 对应倾向：认真上分、讨厌摆烂
- B. 对面节奏太快，全程被针对
  - 对应倾向：carry压力、打野/对抗倾向
- C. 队友互相甩锅，气氛变差
  - 对应倾向：重视团队氛围
- D. 自己突然没手感，操作变形
  - 对应倾向：自我要求高、操作型

题目 5：队友坑崩整局时，你通常会？
- A. 不喷不骂，认真打完
  - 对应倾向：心态、队友好感度
- B. 沉默处理，打完就下一局
  - 对应倾向：安静、独狼
- C. 忍不住吐槽两句，但不会真吵
  - 对应倾向：直爽、真实
- D. 心态爆炸，想立刻下线冷静
  - 对应倾向：摆烂指数、情绪化

题目 6：你打王者的核心目的是？
- A. 认真冲分，想变得更强
  - 对应倾向：carry、认真
- B. 放松娱乐，开心最重要
  - 对应倾向：整活、佛系
- C. 陪朋友一起玩，有人开黑才有意思
  - 对应倾向：陪伴、固排
- D. 解压消遣，打几局换换心情
  - 对应倾向：随缘、解压

## 六、评分维度与规则

### 6.1 五大评分维度

- 操作分
- 意识分
- 心态分
- 摆烂指数
- 队友好感度

### 6.2 评分规则概述

1. 选择题结果作为基础分值。
2. 基础信息进行加权调整：
   - 本命英雄影响操作、意识、队友好感度、人设标签；
   - 常用分路影响操作、意识、carry 倾向、团队倾向；
   - 游戏性格影响心态、摆烂指数、队友好感度；
   - 组队状态影响队友好感度和人设匹配；
   - 游戏 ID 仅用于趣味标签，不进入核心评分。
3. 综合后生成五维评分与人设标签，确保结果既符合用户输入，也保持轻松趣味。

### 6.3 评分生成机制

- 每题选项映射到一个或多个维度的基础分。
- 不同输入项按权重叠加到对应维度：
  - 例如“打野”更偏向意识与 carry 倾向，“温柔稳局型”提升队友好感度与心态分，“常年单排”增加独狼倾向但降低队友好感度。
- 最终评分应保持可视化区间，例如 0-100 分或 0-10 分段，便于展示结果卡片。

### 6.4 结果平衡原则

- 不因单一题目或单一输入极端倾向导致结果失衡。
- 游戏 ID 仅输出趣味标签，不做分数基础。
- “第一次见面英雄”只在最终文案礼物彩蛋中体现，不影响五维评分核心逻辑。

## 七、人设匹配与评语逻辑

根据现有大纲，提供 6 套人设评语。人设匹配基于“游戏性格+组队状态+分路倾向+题目选择风格+本命英雄类型”等综合因素。

### 7.1 人设类型

人设 1：温柔稳局陪玩型
- 适配条件：游戏性格：温柔稳局型 / 佛系随缘型；组队状态：固定双排 / 陪朋友上线 / 三五排开黑；分路：游走 / 中单较适配。

人设 2：高冷 Carry 实力型
- 适配条件：分路：打野 / 对抗路 / 发育路；游戏性格：安静发育型 / 认真指挥型；目的：认真冲分。

人设 3：峡谷整活气氛型
- 适配条件：游戏性格：整活气氛型；目的：放松娱乐；摆烂指数偏高但心态分不低。

人设 4：安静独狼单排型
- 适配条件：组队状态：常年单排 / 随缘组队；游戏性格：安静发育型；题目倾向：沉默、独立、稳健。

人设 5：直爽真实上分型
- 适配条件：游戏性格：直爽吐槽型 / 认真指挥型；题目中选择“沟通”“吐槽”“认真打”的比例较高；心态分中等，操作分或意识分较高。

人设 6：专属开黑搭子型
- 适配条件：组队状态：固定双排 / 三五排开黑 / 陪朋友上线；目的：陪朋友一起玩；队友好感度高。

### 7.2 评语范例

每个人设应输出一段专属评语，并融入用户输入的关键点（本命英雄、分路、游戏 ID 关键词、生日语气）。

例如：
- 人设 1 可以强调“稳局型玩家、顺风不乱浪、逆风不急躁、队友心态拉回来”。
- 人设 2 可以强调“话不多但存在感强、该发育时发育、关键时刻能站出来打出存在感”。
- 人设 3 可以强调“你打游戏的重点不只是输赢，还有快乐本身、整活气氛、名场面”。
- 人设 4 可以强调“独自闯峡谷、低调自洽、不喜欢把情绪丢给别人”。
- 人设 5 可以强调“直爽真实、不故意制造矛盾、对对局比较投入、胜负心强”。
- 人设 6 可以强调“适合长期组队、靠谱有回应、愿意陪朋友一起打”。

## 八、生日彩蛋与抽签机制

### 8.1 运势抽签

- 在完成选择题后进入“抽签”环节。
- 抽签结果应与用户画像和人设评语形成联动，但不必统计分数，仅作为趣味祝福。
- 抽签可展示 3-4 条签文，例如“红签、黄签、蓝签、祝福签”，其中祝福签带有生日专属祝词。

### 8.2 生日彩蛋规则

- 生日彩蛋应嵌入结果页，结合“第一次见面英雄”“游戏 ID”“本命英雄”等信息，生成专属化的语句。
- 例如对答对“第一次见面英雄”的用户，额外输出一句“你还记得我们第一次见面时的那个英雄，说明这份峡谷友谊真的很稳”。
- 对答错的用户，输出“虽然记错了，但后来我们一起玩的每一局更重要”。

## 九、技术实现方案

### 9.1 前端架构

- 推荐实现方式：静态前端页面，单页应用或多页单问题流程。
- 技术选型建议：纯 HTML/CSS/JavaScript 或 React/Vue 轻量实现，均可。
- 组件划分：
  - 引导页 / 开始页
  - 填写信息页（按顺序展示单题）
  - 选择题页（每页一个问题）
  - 抽签页
  - 结果页（评分卡 + 人设评语 + 生日彩蛋）

### 9.2 页面流程控制

- 每一页只展示一个问题，用户完成当前答案后进入下一页。
- 禁止后退按钮返回前一题；如果用户尝试后退，可提示“已进入下一阶段，当前结果不可回退”。
- 页面间状态在前端存储：可用内存状态管理、local state 或临时 session 数据。

### 9.3 数据驱动配置

建议将题目、选项、权重、人设模板、英雄标签、ID 关键词等信息放在 JSON / JS 配置结构中，便于后续调整。

数据模型示例：
- `questions`: 所有题目与选项、对应倾向编码
- `heroProfiles`: 本命英雄与对应分值调整
- `laneProfiles`: 分路与对应分值调整
- `personalityProfiles`: 游戏性格与对应评分/标签调整
- `teamProfiles`: 组队状态与评分/标签调整
- `idKeywords`: ID 关键词与趣味标签映射
- `personas`: 人设匹配规则与评语模板

### 9.4 结果生成逻辑

- `score = base + inputAdjustment`
- `base` 来自 6 道选择题的答案映射
- `inputAdjustment` 来自本命英雄、常用分路、游戏性格、组队状态
- 将最终结果归一化为可展示范围；同时产生人设标签与祝福文案
- `第一次见面英雄`生成额外彩蛋文案，不参与评分

### 9.5 可部署方式

- 优先静态托管：GitHub Pages、Netlify、Vercel、静态对象存储。
- 如果需要收集数据或分享记录，可后续接入轻量后端 API，但初版应以静态前端为主。

## 十、UI / UX 规范

### 10.1 风格要求

- 简单轻盈，避免过于复杂的卡牌或重度游戏 UI。
- 使用生日礼物、彩带、蛋糕、气球、闪光、生日蜡烛等元素作为装饰。
- 颜色宜以暖色系和节日氛围为主，例如粉色、橙色、金色、淡紫、浅蓝。
- 页面结构保持单一问题聚焦，卡片式输入框、简洁按钮、清晰进度提示。

### 10.2 交互规则

- 每页仅显示一个问题。
- 用户回答后立即跳转下一题，不能回退。
- 提交时给出轻快反馈，例如“下一步”“继续闯关”“抽签去！”。
- 结果页应有明显生日气氛提示，如“生日快乐”、“专属祝福”、“今日专属峡谷人设”。

### 10.3 视觉表现

- 进度指示：例如“问题 3/9”、“测评进行中”“生日专属阶段”等。
- 动画细节：按钮点击、页面过渡可使用轻微渐变和弹性动画；抽签环节可用翻牌或滚动效果。
- 结果呈现：评分卡、人物标签、人设评语、签文与生日彩蛋应分块展示，避免信息堆叠。

## 十一、非功能需求

- 移动优先：优先适配手机竖屏，确保单题模式在小屏幕上交互顺畅。
- 性能：无需外部依赖时尽量减少包体积，保持页面快速加载。
- 可访问性：按钮、文本框具备可点击区域与键盘导航，结果文案易读。
- 分享性：结果页应支持快速复制或分享链接/截图提示。
- 可配置性：题目、权重、人设说明可在配置文件中调整，以便后续迭代。

## 十二、验收标准

- 流程完整：用户完成“填信息→答题→抽签→评分→人设评语→生日彩蛋”闭环。
- 单题交互：页面仅展示一个问题，问题完成后才能进入下一页，且不能返回修改。
- 评分逻辑：`本命英雄`、`常用分路`、`游戏性格`、`组队状态` 与选择题共同决定结果，`游戏 ID` 仅作为趣味标签，`第一次见面英雄` 仅作为隐藏彩蛋。
- 视觉风格：页面整体简单轻盈，存在生日主题元素。
- 结果体验：用户最终能看到五维评分、人设评语和生日祝福文案，且文案中出现专属化描述。

## 十三、实施建议与迭代

### 13.1 初版建议

- 先完成静态前端版本，按流程实现单题模式与结果生成逻辑。
- 先用固定数据与模板生成结果，后续再迭代引入 AI 文案或更多个性化分支。

### 13.2 版本迭代方向

- 增加更多本命英雄标签与语义匹配规则。
- 让抽签结果更加个性化，例如与分数或人设联动。
- 添加分享卡片或海报生成功能。
- 加入好友祝福录入、多人比拼等社交扩展。
